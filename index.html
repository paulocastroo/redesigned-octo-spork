tileAction(tileContent: TileContent): void {

Part 20: More optimization

· Navigating to a feature page and going back to dashboard always triggers the http request to get our DashboardContent json. We can improve this by saving the result of the request to the state of the app

37. Edit content-provider.service.ts

37.1 Add two properties

private content = [];

private contentObs: Observable<any>;

37.2 Import two modules from rxjs

import 'rxjs/add/observable/of';

import 'rxjs/add/operator/map';

37.3 Modify function getContentByEndpoint()

getContentByEndpoint(endpoint: string): Observable<any> {

  if (this.content[endpoint]) {

    return Observable.of(this.content[endpoint]);

  }

  this.contentObs = this._http.get(endpoint).map(response => {

    this.contentObs = null;

    this.content[endpoint] = response;

    return response;

  });

  return this.contentObs;

}

· The first if checks if the content by this endpoint is already existing if true, return the value as an Observable using the rxjs of so we can still subscribe into it

· The latter part of the function is to make an http request

· You can now check that navigating through our app immediately show the content on the dashboard unless the page was reloaded, closed or a redirection happens
